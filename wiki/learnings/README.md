# Operative learnings

One page per topic (`<topic>.md`, kebab-case), updated by the knowledge-commit step
whenever a run produced reusable experience. This is where the system learns what works
for THIS project — and where **own evidence outranks persona defaults**.

## Format

```markdown
---
type: learning
created: YYYY-MM-DD
updated: YYYY-MM-DD
scope: <where this applies: roster | youtube | email | ...>
---

# <Topic>

## Persona view
What the clones recommend, attributed and cited to their wikis.

## Own evidence
What OUR runs actually showed. Dated entries with sample size and outcome.
- [YYYY-MM-DD] <observation> (n=..., source: <artifact/log entry>)

## Effective rule
The rule currently in force for this project, derived from the above. When it diverges
from the persona view, say so and why. Councils load this section into every seat's
brief.

## Open questions
What would change the rule; what to measure next.
```
